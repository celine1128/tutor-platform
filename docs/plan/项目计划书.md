## 一、项目概述

### 1. 项目背景

随着家长对学生课外辅导的需求增加，以及大学生勤工俭学的意愿增强，校园周边的家教信息往往分散在微信群、QQ群中，存在信息杂乱、匹配效率低、缺乏规范流程等问题。本项目基于Java Web技术开发一个轻量级的家教管理平台，实现需求发布、接单、订单确认等核心功能，方便家长与大学生家教直接对接。

### 2. 项目目标

开发一套完整、稳定、易操作的家教管理平台，支持用户注册/登录、家长发布家教需求、老师查看需求并接单、双方查看订单、家长确认完成等功能，达到课程设计要求。

### 3. 开发环境与条件

- **开发工具：** IntelliJ IDEA
- **后端技术：** Java + Spring Boot
- **前端界面：** HTML + Thymeleaf + Bootstrap
- **数据库：** MySQL 8.0
- **版本控制：** Git + GitHub
- **运行环境：** Windows，JDK 17

## 二、需求分析

### 1. 目标用户与使用场景

- **家长用户：** 注册登录后，发布家教需求（科目、年级、价格、描述），查看自己发布的需求列表，在订单中确认完成。
- **老师用户（大学生）：** 注册登录后，查看所有待接单的需求，点击接单，查看自己的订单列表。
- **系统不设复杂管理员后台**，基础数据可通过数据库直接维护。

### 2. 核心功能清单

- 用户注册/登录（手机号、密码、角色选择）
- 家长：发布需求（科目、年级、价格、描述）
- 家长：查看我已发布的需求列表（显示待接单/已接单状态）
- 老师：查看所有待接单的需求列表
- 老师：点击“接单”按钮（生成订单，需求状态变为已接单）
- 双方：查看订单列表（家长看到自己发布的订单，老师看到自己接的订单）
- 家长：在订单列表中点击“确认完成”（订单状态变为已完成）

### 3. 非功能需求

- **性能：** 页面加载 ≤ 2秒，操作响应及时
- **安全：** 密码加密存储（BCrypt），未登录不能访问功能页面
- **兼容：** 支持Chrome浏览器，页面简洁可用

## 三、技术路线

### 1. 总体架构

B/S架构（浏览器/服务器），采用前后端不分离方式。Spring Boot 作为后端框架，Controller 直接返回 Thymeleaf 模板页面，使用 Session 保持登录状态，降低开发复杂度，便于小团队快速交付。

### 2. 主要技术

- **前端：** Thymeleaf + Bootstrap 5（CDN），少量原生JavaScript
- **后端：** Spring Boot + MyBatis-Plus
- **数据库：** MySQL 8.0
- **认证：** Session（不用JWT）

### 3. 数据设计初步考虑

- **主要实体：** 用户、家教需求、订单
- **关键数据表（3张）：**
  - `user`（用户表）：id、手机号、密码、角色（parent/teacher）、昵称
  - `demand`（需求表）：id、家长id、科目、年级、价格、描述、状态（0-待接单，1-已接单）、发布时间
  - `order`（订单表）：id、需求id、老师id、家长id、状态（0-待确认，1-已完成）、创建时间、完成时间
