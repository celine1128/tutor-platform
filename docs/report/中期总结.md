# 家教管理系统 项目中期工作总结

## 一、项目信息
项目名称：家教管理系统

## 二、项目整体概况
本阶段团队四人分工协作，依次完成**用户账号权限、家长家教需求、教师订单业务、前后端架构与UI规范化**全流程开发。完成系统登录认证、需求发布查询、教师订单管理、前端页面搭建、项目环境统一、架构版本适配、Git版本管控等全部核心工作，项目架构稳定、功能完整，整体进度符合中期开发计划。

## 三、团队全员已完成工作内容
### （一）用户账号注册登录与权限管控模块
- 完成学生、教师角色通用注册接口，支持手机号、密码、角色、昵称录入，实现手机号唯一校验，采用BCrypt加密存储密码，保障账号安全。
- 开发通用登录、获取当前用户信息、退出登录全套接口，基于Session维护登录状态。
- 自定义HandlerInterceptor登录拦截器，统一配置接口放行规则，未登录请求返回401状态码，实现全局身份权限校验。
- 完成用户表数据库设计，严格遵循Controller、Service、Mapper、Entity四层分层架构，使用MyBatis Plus LambdaQueryWrapper高效条件查询。

### （二）家长端家教需求管理模块
- 独立完成家长专属登录身份校验，仅合规家长账号可访问家教相关业务模块。
- 实现家教需求发布功能，支持学科、年级、上课时间、地点、薪资等信息填写，完成参数校验与数据库持久化存储。
- 开发个人需求列表查询接口，支持时间排序、状态筛选，方便家长管理自身发布订单。
- 完成家长用户表、家教需求表建表脚本编写，代码规范整洁，全部提交至dev分支统一管理。

### （三）教师业务与订单管理模块
- 完成TeacherController教师接口、OrderController订单列表相关接口开发。
- 实现核心业务逻辑编写，并完成全量接口自测与调试。
- 修复Git提交身份异常问题，重新配置本地用户信息，规范代码提交归属，成功推送至远程仓库。
- 排查修复接口路径、请求参数不匹配问题，通过日志调试定位BUG，保障接口正常调用。

### （四）项目架构重构、前端UI与环境统一统筹
- 基于Bootstrap5完成首页、登录页、注册页UI重构，搭建统一全站视觉风格，实现响应式页面布局。
- 使用Thymeleaf封装通用导航栏、页脚公共组件，实现页面组件复用，统一全站样式。
- 统一项目JDK17开发环境，解决版本不兼容、编译冲突问题，规范团队开发环境。
- 优化项目目录结构、规范pom.xml配置，梳理Maven依赖，接入阿里云镜像，解决依赖下载失败、构建异常问题。
- 完成SpringBoot3.x Jakarta EE全版本包迁移，修复javax转jakarta相关报错，适配新版拦截器逻辑。
- 重构登录拦截器，适配新版框架，稳定实现Session校验与页面重定向。
- 规范Tomcat本地运行配置，保障localhost:8080服务正常启动、业务链路全连通。
- 优化Git仓库文件过滤，保持仓库代码整洁，规范团队提交流程，保障多人顺畅联调。

## 四、项目开发问题及解决方案
1.  **Git提交作者信息错误**：重新配置本地Git用户名与邮箱，修正身份归属，正常推送远程仓库。
2.  **接口路径、参数匹配异常**：通过后台日志断点调试，逐层排查接口逻辑，快速修复调用异常。
3.  **框架版本升级兼容报错**：完成javax→jakarta全量包迁移，重写适配新版拦截器逻辑。
4.  **多人环境不一致编译失败**：统一JDK版本、规范Maven依赖、更换阿里云镜像，解决环境冲突。
5.  **项目目录混乱、IDEA识别异常**：重构项目结构，规范pom.xml位置，恢复Maven正常加载。
