# 家教管理平台 - 需求分析文档
## 1. 项目概述
本项目为基于 SpringBoot + Thymeleaf + MySQL 开发的**家教线上对接管理平台**，主要解决家长发布家教需求、大学生老师在线接单的线下信息不对称、对接效率低的问题，实现需求发布、接单、订单确认全流程线上化，适用于课程设计、小型家教服务场景。

## 2. 运行环境
- 后端：SpringBoot 2.7.15、MyBatis‑Plus 3.5.3.1
- 前端：Thymeleaf、Bootstrap 3
- 数据库：MySQL 8.0
- JDK：1.8
- 服务器：内置 Tomcat

## 3. 用户角色
系统包含两类核心用户：
1. **家长**：发布家教需求、查看个人需求、查看订单、确认订单完成
2. **大学生老师**：查看待接单需求、在线接单、查看个人订单

## 4. 功能需求
### 4.1 公共功能模块
1. 用户注册：手机号、密码、选择身份（家长/老师），密码 MD5 加密存储
2. 用户登录：手机号+密码校验，区分角色跳转对应首页
3. 退出登录：销毁 Session，返回登录页
4. 权限拦截：未登录禁止访问系统功能；家长、老师角色页面互相隔离，不可越权访问

### 4.2 家长端功能
1. 发布家教需求：填写辅导科目、年级、时间、地点、期望时薪
2. 查看我的需求：查看本人发布的所有需求，区分**待接单、已接单**状态
3. 查看我的订单：查看老师接单后生成的订单
4. 确认订单完成：辅导结束后，确认订单状态变更为已完成

### 4.3 老师端功能
1. 浏览待接单需求：仅查看状态为**待接单**的家长需求
2. 在线接单：对有效需求接单，自动生成订单，需求状态变更为已接单
3. 查看我的订单：查看本人已接单的所有订单，区分**待确认、已完成**

## 5. 非功能需求
1. 安全性：密码加密存储、角色权限隔离、防重复接单、防重复确认
2. 易用性：界面简洁，操作流程清晰，适配电脑端浏览器
3. 稳定性：订单、需求状态流转正常，数据持久化存储
