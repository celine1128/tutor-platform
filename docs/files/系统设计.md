# 家教管理平台 - 系统设计文档
## 1. 总体架构
本系统采用经典 **MVC 三层架构**：
- 表现层（Controller）：接收前端请求，页面跳转、参数传递
- 业务层（Service）：实现业务逻辑、状态校验、权限控制
- 数据访问层（Mapper）：基于 MyBatis‑Plus 操作 MySQL 数据库
- 前端：Thymeleaf 模板引擎 + Bootstrap 实现页面渲染

## 2. 系统模块划分
1. 用户模块：注册、登录、权限拦截、退出登录
2. 家长模块：需求发布、我的需求、订单确认
3. 老师模块：需求浏览、接单、我的订单
4. 工具模块：MD5 密码加密工具

## 3. 数据库设计
### 3.1 数据表结构（共3张）
#### （1）sys_user 用户表
| 字段 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| id | int | 主键自增 | 用户ID |
| phone | varchar(20) | 唯一、非空 | 手机号 |
| password | varchar(50) | 非空 | MD5加密密码 |
| role | varchar(10) | 非空 | 角色：parent/teacher |

#### （2）tutor_demand 家教需求表
| 字段 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| id | int | 主键自增 | 需求ID |
| parent_id | int | 非空 | 发布需求的家长ID |
| subject | varchar(20) | | 辅导科目 |
| grade | varchar(20) | | 辅导年级 |
| time | varchar(50) | | 辅导时间 |
| location | varchar(100) | | 辅导地点 |
| salary | decimal(10,2) | | 时薪 |
| status | int | 默认0 | 0待接单，1已接单 |

#### （3）tutor_order 订单表
| 字段 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| id | int | 主键自增 | 订单ID |
| demand_id | int | 非空 | 关联需求ID |
| parent_id | int | 非空 | 家长ID |
| teacher_id | int | 非空 | 接单老师ID |
| status | int | 默认0 | 0待确认，1已完成 |

### 3.2 核心业务流程
1. 家长流程：注册→登录→发布需求→等待老师接单→查看订单→确认完成
2. 老师流程：注册→登录→查看待接单需求→接单→生成订单→等待家长确认
3. 状态流转：需求（待接单→已接单）→订单（待确认→已完成）
